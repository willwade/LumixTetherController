1. About This Document

1.1. Introduction

This document describes the APIs using the Lumix Remote Control Library.

The Lumix Remote Control Library is provided in DLL (Lmxptpif.DLL) format.

  • System Requirements of Lmxptpif.DLL
    • Compatible OS: Windows 10 64bit
  • Development Environment
    • Microsoft™ Visual Studio 2015
    • At runtime, Visual Studio 2015 runtime library must be installed

Related files for this library

File name File description
"Lmx_lib_func_api.h" API header provided with Lmxptpif.dll
"Lmx_lib_type.h" Header indicating the format of variables used in this library
"Lmx_lib_def.h" Header containing a collection of various definitions used in this library, such as constant definitions and structure definitions
"Lmx_lib_dev_common.h" Header for selecting connected device to be used with this library
"lmx_lib_plat.h" Header for switching the environment used in this library
"Lmxptpif.lib" Link library for using Lmxptpif.dll
"Lmxptpif.dll" The actual communication library used with this library

This document is consisted of four chapters.
Chapter 1: Describes this document.
Chapter 2: Describes the API specifications that can be used with this SDK.
Chapter 3: Describes the structure definitions used in this SDK.
Chapter 4: Describes the value definitions used in this SDK.

2. API Specifications

2.1. Library initialization

This section explains the API for initialization when using the communication library.

2.1.1. LMX_func_api_Init

Initializes the communication library.

[Format]

void LMX_func_api_Init ( void )

[Parameters]

None

[Return values]

None

[Notes]
This API is used to initialize the communication library when using the communication library. This API needs to be called first when using the communication library.

2.2. Device connection and disconnection

This section explains API related to devices, such as acquisition of devices to be connected, connection of selected devices and disconnection of connected devices.

2.2.1. LMX_func_api_Get_PnPDeviceInfo

Gets information on devices that can be connected.

[Format]

UINT8 LMX_func_api_Get_PnPDeviceInfo ( PLMX_CONNECT_DEVICE_INFO plmxPnpDevInfo )

[Parameters]

PLMX_CONNECT_DEVICE_INFO plmxPnpDevInfo

Pointer of LMX_CONNECT_DEVICE_INFO structure for storing information on devices that can be connected.

[Return values]

UINT8

LMX_BOOL_TRUE is returned when information on devices that can be connected is got successfully.

2.2.2. LMX_func_api_Select_PnPDevice

Performs the connection process of a selected device.

[Format]

UINT8  LMX_func_api_Select_PnPDevice (
    UINT32                   dwDevIndex,
    PLMX_CONNECT_DEVICE_INFO plmxPnpDevInfo
)

[Parameters]

UINT32 dwDevIndex

Index value indicating the location where the selected device's information is stored.

PLMX_CONNECT_DEVICE_INFO plmxPnpDevInfo

Pointer of LMX_CONNECT_DEVICE_INFO structure for storing information on devices that can be connected.

[Return values]

UINT8

LMX_BOOL_TRUE is returned when information on devices that can be connected is got successfully.

[Notes]
Information on devices that can be connected needs to be got using LMX_func_api_Get_PnPDeviceInfo before processing this API..

2.2.3. LMX_func_api_Close_Device

Performs the disconnection process for a connected device.

[Format]

UINT8   LMX_func_api_Close_Device ( void )

[Parameters]

None

[Return values]

UINT8

LMX_BOOL_TRUE is returned when the disconnection process of the connected device is completed normally.

[Notes]
A device needs to be connected using LMX_func_api_Select_PnPDevice before processing this API.

2.2.4. LMX_func_api_Open_Session

Opens a session to use USB.

[Format]

UINT8  LMX_func_api_Open_Session ( 
    UINT32  ulConnectVer,
    UINT32* pulDeviceConnectVer
)

[Parameters]

UINT32 ulConnectVer

Connected version.
The upper 16 bits mean the major version of SDK according to the definition of LMX_CONNECT_MAJOR_VERSION.
The lower 16 bits mean the minor version of SDK according to the definition of LMX_CONNECT_MINOR_VERSION.
As of now this value is designated as 0x00010001.

UINT32* pulDeviceConnectVer

Specifies the address of the variable to store the connected version information.
Declare the variable in the application to store the connected version information.

[Return values]

UINT8

LMX_BOOL_TRUE is returned when start a session with USB device successfully.

2.2.5. LMX_func_api_Close_Session

Closes a session.

[Format]

UINT8  LMX_func_api_Close_Session ( void )

[Parameters]

None

[Return values]

UINT8

LMX_BOOL_TRUE is returned when close a session with USB device successfully.

2.3. Callback function registration

This section explains the callback function, which notifies the application when the remote control library gets an event.

2.3.1. LMX_func_api_Reg_NotifyCallback

Register a callback function for notifications from the remote control library to the application when a change in the device occurs.

[Format]

UINT32 LMX_func_api_Reg_NotifyCallback (
    UINT32            CallBackType,
    LMX_CALLBACK_FUNC appfunc
)

[Parameters]

UINT32 CallBackType

Specifies the callback type.
See Callback type for information on callback types.

LMX_CALLBACK_FUNC appfunc

Specifies the address of the callback function associated with the callback type.

[Return values]

UINT32

LMX_DEF_ERR_NO_ERROR is returned when a callback function is registered successfully.

[Notes]
The same callback function can be set for multiple callback types, but a specific callback type cannot be set for multiple callback functions.

2.3.2. LMX_func_api_Delete_CallBackInfo

Deletes the registration of the callback function for sending notifications to the application from the communication library.

[Format]

UINT32 LMX_func_api_Delete_CallBackInfo (
    UINT32 CallBackType
)

[Parameters]

UINT32 CallBackType

Specifies the callback type.
See Callback type for information on callback types.

LMX_CALLBACK_FUNC appfunc

Specifies the address of the callback function associated with the callback type.

[Return values]

UINT32

If registration of the callback function is deleted successfully, LMX_DEF_ERR_NO_ERROR is returned.

2.3.3. LMX_CALLBACK_FUNC

Describes the formats of callback functions registered in LMX_func_api_Reg_NotifyCallback.

[Format]

void  __stdcall Registered_callback_function (
    UINT32 cb_event_type,
    UINT32 cb_event_param
)

[Parameters]

UINT32 cb_event_type

Stores the notified callback type.
See Callback type for information on callback types.

UINT32 cb_event_param

Stores the parameter information associated with the callback type.

[Return values]

None

[Notes]
See the next section for information on how to get associated information for each callback type.
SendMessage in WindowsAPI cannot be used in callback functions.
Use PostMessage to perform notification processes from a callback function to an application.

4.4. Value definition for shutter speed

4.4.1. Shutter speed value

When Shutter speed value is 1 second or more:
Shutter speed setting value is the actual shutter speed value multiplied by 1000 plus 0x8000000.
(However, Bulb, Auto and unknown are excluded.)

Example) When the actual shutter speed value is 30 seconds, shutter speed setting value becomes 0x8000C350. (30x1000=30000(0x0000C350h))
Example) When the actual shutter speed value is 1.6 seconds, shutter speed setting value becomes 0x80000640. (1.6x1000=1600(0x00000640h))

When Shutter speed value is less than 1 second:
Shutter speed setting value is the denominator value of actual shutter speed value multiplied by 1000.
(However, Bulb, Auto and unknown are excluded.)

Example) When the actual shutter speed value is 1/3.2 seconds, shutter speed setting value becomes 0x00000C80. (3.2x1000=3200(0x00000C80h))
Example) When the actual shutter speed value is 1/30 seconds, shutter speed setting value becomes 0x0000C350. (30x1000=30000(0x0000C350h))
